Отзыв о финальном проекте
Антона Шелина
Школа Высшей Математики
Автор проекта поставил перед собой задачу создать помощника для написания
кода Component Pascal и BlackBox. Для решения поставленной задачи им был
собран датасет, дообучена модель (SFT техника) Qwen Coder 2.5 Instruct 3B. Также
были проведены эксперименты с меньшими моделями.
К сильным сторонам проекта можно отнести:
1. Собран уникальный датасет по языку Component Pascal и BlackBox.
2. Дообученная модель способна генерировать запускающийся код чаще, чем
существующие модели и сервисы (OpenAI ChatGPT, DeepSeek V3).
3. Использованы современные техники и нетипичный выбор модели: студент
использует SFT подход и модель из семейства Qwen, вместо более
распространенного подхода с обучением BERT или небольших GPT.
4. Заметна высокая степень автоматизации проекта и использование
нескольких ЯП при его реализации.
5. Грамотный подход к обучению модели, продемонстрировано понимание
алгоритма действий при различных проблемах при дообучении.
6. RAG с использованием pgvector.
7. Сервис реализован с мониторингом, используется Grafana + Prometheus.
8. Реализован бот в Telegram.
9. Реализован фронтенд с использованием Web Assembly.
К недостаткам проекта можно отнести:
1. Общая неопрятность репозитория. Советую использовать инструменты по
типу pre-commit и nbqa для поддержания стиля кода и не только. Следовало
бы также удалять длинные комментарии (например, из docker-compose),
или же описать зачем они могут быть использованы. Большинство Jupyter
Notebooks обладают минимальными комментариями, а некоторые не
обладают и названием (Untitled.ipynb в /learn/notebooks).
2. Недостаток документации проекта. Например, для хостинга предоставлен
docker-compose.yml, но не предоставлены инструкции по его запуску (явно
требуется указывать переменные окружения), а также файлы с примерами
содержания по типу db_password.txt.example. В целом, инструкций по
запуску парсера для форумов и прочего тоже нет, хотя следовало бы
добавить.
3. Деградация качества модели при квантизации описана лишь «gguf q8
модель работает хуже чем родная hf модель.». Стоит измерить хотя бы
стандартные для генерации текста, но можно попробовать и заняться более
применимыми для задачи генерации кода CodeBLEU или в какой-то
степени METEOR (при возможности их использования на выбранных ЯП).
4. Отсутствие описания проведенных экспериментов.
5. Опечатки в тексте, в том числе в презентации.
6. Несмотря на определенные успехи в создании фронтенда, сам по себе он
довольно сырой и требует улучшений.
В целом, итоговый проект Антона Шелина является примером хорошего проекта.
Недостатки в основном косметические, и к функционалу модели относятся в
меньшей степени. При должных усилиях в формировании документации проект
станет прекрасным примером AI ассистента для не очень популярной системы и
языка программирования. Функционал Telegram бота уже достаточен для
использования первыми пользователями.
Наибольшей проблемой проекта кажется низкое качество собранных данных,
улучшение которых требует, в том числе, разметки человеком. Заметно желание
автора использовать малые модели, доступные для запуска на ноутбуках у
школьников, в помощь которым и предлагается использовать данный чат-бот-
аналог Copilot.
В заключении хочется пожелать автору довести до какого-то промежуточного
конца данный проект в смысле кода, привести в порядок документацию и
написать о нём заметку на Habr или профильные по системе BlackBox и
Component Pascal форумы, поскольку кажется, что потенциал здесь очень
высокий.
